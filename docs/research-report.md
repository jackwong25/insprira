## 二、项目本质

**基于 RedFox 付费 API 的「本地自媒体数据中枢」** —— 把抖音、小红书、公众号等 9+ 平台的热榜/账号/爆款数据聚合到一个私有 Web 工作台，叠加上 LLM 辅助选题、知识库对接、违禁词检测、AI 封面生成。

```
┌─────────────────────────────────┐
│          Insprira UI            │  ← 本地 Web 界面 (Tailwind CSS)
│  (热榜 | 选题 | 账号 | 知识库) │
├─────────────────────────────────┤
│        server.js (5,834行)      │  ← Node.js 单体后端
│  API代理 · SQLite缓存 · LLM调度 │
├─────────────────────────────────┤
│       RedFox API (付费)         │  ← 第三方数据源
│  抖音/小红书/公众号/B站/快手... │
└─────────────────────────────────┘
```

## 三、核心功能模块

### 1. 热榜与趋势
- 全网热点聚合（7 平台：抖音/微博/B站/快手/知乎/头条/百度）
- 平台垂直热榜：抖音 / 小红书 / 公众号 / AI公众号 / AI B站 / AI小红书 / AI抖音 / AI快手 / AI视频号 / 短剧抖音 / 短剧公众号
- 7/14 天趋势分类：增长 ↑ / 稳定 → / 冷却 ↓
- 动态 Tab 机制：通过 Skill 绑定动态扩展热榜平台

### 2. Skill 中心
- 从 redfox-community/skills 拉取并热更新
- LLM 自动分类（热点/创作/分析/检索/生成工具）
- 热点 Skill 一键绑定到热榜 Tab

### 3. 选题生成
- 基于本地热榜证据 + 公众号爆款搜索
- 调用 OpenAI 兼容 LLM 生成选题
- 支持一键加入灵感库

### 4. 账号追踪
- 分组订阅抖音/公众号/小红书账号
- 自动同步作品 + 数据快照
- 多平台账号诊断 + 趋势图表（粉丝/红狐指数/评分/作品数）

### 5. 知识库
- Obsidian 本地 vault 接入
- Notion 云笔记接入
- WeRss 公众号文章 RSS 同步
- AES-256-GCM 加密存储凭证

### 6. 内容重构
- 多平台改写（适配不同平台调性）
- RedFox `gpt-image-2` 封面生成
- 违禁词检测（抖音/小红书/公众号标准）

### 7. 本地 Agent 集成
- 支持 5 种 CLI Agent：Codex / Claude Code / Kimi / OpenClaw / Hermes

### 8. CRON 调度
- 内置：每日热榜快照 / 缓存清理 / WeRss 同步 / 账号追踪刷新
- 支持自定义任务 + 拖拽排序

## 四、架构评估

### 技术栈

| 层 | 选型 | 评价 |
|----|------|------|
| 后端 | Node.js ≥20 · CommonJS · 零框架 | 简单直接，无中间件生态 |
| 数据库 | SQLite（better-sqlite3） | 轻量，适合本地/小团队 |
| 前端 | Vanilla JS · SPA 路由 · Tailwind CSS 3.4 | 无框架依赖，零构建 |
| 认证 | scrypt 密码哈希 + Session | 基础够用，无 RBAC |
| 加密 | AES-256-GCM | 知识库凭证保护合规 |
| 容器化 | Docker + GHCR 多架构镜像 | 部署友好 |

### ⚠️ 架构风险

| 风险 | 严重度 | 说明 |
|------|--------|------|
| **单体巨石** | 🔴 高 | server.js 5,834 行，路由/DB/业务全塞一起 |
| **零测试** | 🔴 高 | package.json 有 test 脚本但无测试文件 |
| **无 Issue 跟踪** | 🟡 中 | Issues 404，无法提交 bug |
| **无框架** | 🟡 中 | 自建路由和中间件，缺少生态保护 |
| **AGPL-3.0** | 🟡 中 | 对外提供服务须开源修改 |
| **默认密码** | 🟡 中 | admin/123456，无强制修改 |
| **极新项目** | 🔴 高 | 仅 4 天历史，API 可能随时变动 |
